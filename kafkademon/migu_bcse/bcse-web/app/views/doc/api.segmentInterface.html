2.3 分词接口

1. 基本描述

表2.3.1.1. 基本描述
接口名称 分词体验接口
接口描述 根据用户选择分词字段将用户输入的内容进行分词

2. 请求信息

表2.3.2.1. 请求信息
前提要求 无
请求类型 HTTP
请求地址 /analyzer
请求方式 GET
报文格式
content-type：application/json
accept：application/json
接口参数 ?userId=xxx&appId=xxx&fieldSearch=xxx&searchQuery=xxx

3. 参数说明

表2.3.3.1. 请求参数描述
字段 约束 类型 长度 描述 说明
appId 必填 String 50 应用ID
userId 必填 String 50 用户ID
searchQuery 必填 String 300 要分词内容 最大支持300个字符，当搜索文本长度大于300时，自动截取前300个字符
fieldSearch 必填 String 分词字段 只能是： ANSJ，QP_NoSplit, JP_Nosplit ANSJ_JP ANSJ_QP ANSJ_PY 中的一种 
        
          URL:http://ip:port/bcse/analysis?
          userId=chenmin@mcss.chinamobile.com&appId=1&fieldSearch=ANSJ&searchQuery=xxx

4. 响应消息

表2.3.4.1. 返回参数描述
字段 父节点 约束 类型 长度 描述 说明
analysis result Array 结果列表 
        
          {
              "message": "操作成功",
              "code": "",
              "status": "SUCCESS",
              "result": {
                  "analysis": [
                      "你是谁",
                      "，",
                      "来自",
                      "哪里"
                  ]
              }
          }
