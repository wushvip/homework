2.2 搜索结果接口

1. 基本描述

表2.2.1.1. 基本描述
接口名称 搜索结果接口
接口描述 根据用户的输入词到后台检索命中的数据记录并返回相应数据

2. 请求信息

表2.2.2.1. 请求信息
前提要求 无
请求类型 HTTP
请求地址 /search
请求方式 POST
报文格式
content-type：application/json
accept：application/json
接口参数

3. 参数说明

表2.2.3.1. 请求参数描述
字段 约束 类型 长度 描述 说明
appId 必填 String 50 应用ID
userId 必填 String 50 用户ID
searchQuery 必填 String 100 搜索文本 最大支持100个字符，当搜索文本长度大于100时，自动截取前100个字符
sortRule 可选 String 字段排序 高级搜索：字段排序
sortConfig 可选 String 50 排序配置 规则名称
facetRule 可选 String 分组统计 规则名称
highLightFields 可选 String 高亮
filterJson 可选 Json 分类过滤 分类过滤信息，二维数组形式；根据搜索结果展示是否有分类筛选功能
pageIndex 可选 Int 请求当前页码 动态分页所需的页码数（默认从1开始）
pageNum 可选 Int 展现需要显示条数 动态显示条数（默认10条）最多200条 
        
        URL:http://ip:port/bcse/search
表2.2.3.2. 请求参数规则描述
字段 来源 规则描述
appId 平台
userId 平台
searchQuery 用户
sortRule 可选
1、 排序字段+空格+排序规则（ASC/DESC）
2、 多个排序使用英文逗号分隔
Example: Time DESC,Number ASC（按照Time降序，Time相同的情况下按照Number升序）
sortConfig 平台 平台配置的排序配置规则名
facetRule 平台 平台配置的分组统计规则名
highLightFields 可选
1、 多个高亮字段使用英文逗号分隔
Example: Name,Title（按照Name,Title高亮显示）
filterJson 可选
1、 每个key:value代表一个过滤条件
2、 多个条件之间是交集
Example:
{ "Number":" [10 TO 100] ", "Number2": " [10 TO 100] " }
这个筛选条件表示取筛选字段Number的值属于Number≥10 && Number≤100并且Number2≥10 && Number2≤100
{ "Number":" {10 TO 100} ", "Number2": " {10 TO 100} " }
这个筛选条件标识取筛选自带Number的值属于Number>10 && Number<100并且Number2>10 && Number2<100
{ "Number":" {10 TO 100] ", "Number2": " [10 TO 100} " }
这个筛选条件标识取筛选自带Number的值属于Number>10 && Number≤100并且Number2≥10 && Number2<100
{ "Number":" [10 TO *] " }
这个筛选条件表示取筛选字段Number的值属于Number≥10
{ "Number":"10 " }
这个筛选条件表示取筛选字段Number的值属于Number=10
{ "Number":"(!10) " }
这个筛选条件表示取筛选字段Number的值属于Number不等于10
